# 项目协作规范

## 代码风格

- 变量名使用小写字母加下划线，例如 `sample_count`。
- 常量名使用全大写字母加下划线，例如 `MAX_ITERATIONS`。
- 类名使用驼峰命名法，例如 `QuantumCircuit`。
- 函数必须编写 docstring，说明函数用途、关键参数和返回值。
- 所有代码注释必须使用中文。

## 数学内容格式

- 在对话输出中的数学内容：行内使用 `\( \)`，行间使用 `\[ \]`。
- 在保存到 Markdown 文件时的数学内容：行内使用 `$ $`，行间使用 `$$ $$`。
- 遇到长公式，尤其是带标签的公式，统一使用行间格式包裹内容。
